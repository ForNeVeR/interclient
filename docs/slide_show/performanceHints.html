A Few Performance Hints

I've started to collect a few performance hints. If you have any more please mail them to me, and maybe we'll publish this along with the rest of our documentation sometime in the future.

  • Use VARCHAR for large text fields
    CHAR strings are fixed length and not compressed by the current interclient remote protocol.
  • Use StringBuffers when concatenating large strings
  • Use the interbase.interclient.Adaptor when stepping thru Date/Times.
  • Use Statement.setFetchSize()
  • Use stored procedures for lots of inserts
  • Use prepared statements for repeated executions of SQL, possibly parameterized.
