Distributing InterClient with your Java Application

The InterClient distribution consists of a set of class files (see Distributed Jar Files) and an InterServer service to be deployed on an InterBase server machine.

The Java archive files, for example interclient.jar, are not digitally signed by the InterBase Software Corporation. For those deploying Java applets, if the web server host (which serves interclient.jar) and the InterServer host are not one and the same machine, then the applet may be digitally signed along with the interclient.jar file for use with a capable browser.

You may also choose to distribute the JRE with your application. See http://www.javasoft.com/products/jdk/1.2/jre
