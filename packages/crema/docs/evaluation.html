Evaluation vs. Full version

Evaluation version

The evaluation version of Crema can be used without restrictions for stand-alone applications.

For applets, the evaluation version is fully functional with the exception of the -ill command line parameter (it is always "on"). As a result, many browsers will refuse to load classes processed with the evaluation version from a server. The following table shows the variety of security policies adopted by some popular browsers:

Browser Loaded from local filesystem Loaded from remote server
AppletViewer works* FAILS
HotJava works* FAILS
Netscape 2.x FAILS FAILS
Netscape 3.x works FAILS
Internet Explorer 3.x works works**

* Class files must be in the current directory!

** Relax. This should not be considered an Internet Explorer security hole. Illegal identifiers are harmless.

The full version of Crema does not have such restrictions; it works with all browsers in all configurations. In addition owners of the full version get free technical support.
